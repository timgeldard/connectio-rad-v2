# Source Verification Coverage Map

> Snapshot: 2026-05-21. The single authoritative reference for the verification state of every known Databricks source object used by ConnectIO RAD V2.
>
> **Important:** "not-verified" means no verification evidence was found in this audit. It does not mean the object is wrong. "not-identified" means no source object has been named for a capability — the object may exist but has not been identified.

## Verification status key

| Status | Meaning |
|---|---|
| `verified-object` | Object confirmed to exist in connected_plant_uat.gold (or named catalog) |
| `verified-schema` | Column list explicitly confirmed (DESCRIBE TABLE or equivalent) |
| `verified-grain` | Primary key / grain confirmed (uniqueness / row-level meaning confirmed) |
| `verified-join` | Join key(s) to other objects confirmed |
| `semantics-pending` | Object+schema verified; field business meaning is unresolved |
| `governance-pending` | Source exists; display rule or business decision required before wiring |
| `not-found` | Explicitly searched; object not present in connected_plant_uat.gold |
| `not-verified` | No verification attempt found in this audit |
| `not-identified` | Source object not yet named for this capability |
| `metric-view-not-row-source` | AI/BI Metric View; not a row-level queryable table |
| `placeholder` | Referenced in code but JOIN/query not yet implemented |

## Biggest verification gaps (prioritised)

1. **SPC `spc_capability_detail_mv` and `spc_nelson_rule_flags_mv` NOT FOUND** — these block SPC capability and rule-signal display; require data platform to apply migrations 012/013.
2. **Quality CoA, deviation, and MIC result source objects not identified** — required before Quality read-only evidence can be wired beyond usage decision.
3. **Warehouse native source views not identified** — routes are wired but SQL table names are not confirmed; schema alignment needed before UAT.
4. **Mass balance semantics (TRACE-P1-010, TRACE-P1-011)** — MOVEMENT_CATEGORY direction mapping and BALANCE_QTY semantics block mass balance UAT.
5. **EnvMon INSPECTION_TYPE filter values unconfirmed** — `'14'` and `'Z14'` not verified in UAT data; may cause empty site-summary responses.
6. **POH view schemas not DDL-verified** — all 4 views are SQL-probed only; full DESCRIBE TABLE not yet run; missing fields (work-centre, timestamps, materialDescription) should be documented from DDL.
7. **gold_batch_lineage LINK_TYPE='DELIVERY' value** — customer exposure route currently filters on this value; not confirmed in live data.
8. **SPC plant-namespace ambiguity (P-prefix vs C-prefix)** — affects mapping between V2 SAP plant codes and SPC-internal plant codes.
