# Field Classification Standard

All fields exposed through the app data layer must be classified to reflect their origin and reliability. If a field sounds like a business decision, it must include a source/governance classification.

## Classifications

| Classification            | Definition                                                                                            | Allowed Use                                                              | Example                               |
| ------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------ | ------------------------------------- |
| **source-field**          | A direct 1:1 mapping from a Databricks source column.                                                 | General use for raw data exposure.                                       | `batchId`, `resultValue`              |
| **source-derived**        | Computed via a deterministic, source-backed SQL or backend algorithm.                                 | General use for established metrics.                                     | `subgroupMean`, `sampleCount`         |
| **application-derived**   | Computed safely on the client side from source fields (e.g. date formatting, simple math).            | Safe presentation logic.                                                 | `daysSinceProduced`                   |
| **application-heuristic** | A best-effort classification or status generated by the application without strict source governance. | Use with caution. Must be paired with a clear `confidenceSource` caveat. | `heuristicQualityStatus`              |
| **governance-pending**    | The field represents a concept that requires a business rule definition which is currently missing.   | Must return a null/unavailable state. The app must not decide.           | `usageDecision` (when rule undefined) |
| **unavailable**           | The data cannot be retrieved or is missing from the underlying model.                                 | To explicitly declare missing data.                                      | `capabilitySource: unavailable`       |
| **not-evaluated**         | A calculation or rule detection has not yet run.                                                      | When a state is unknown because a process hasn't executed.               | `signalsSource: not-evaluated`        |

## Unsafe to Safe Examples

When translating prototype apps to governed data products, risky fields must be reclassified:

- **Unsafe:** `risk: low`
  **Safe:** `risk: unknown` / `riskSource: unavailable`
- **Unsafe:** `recallRecommended: false`
  **Safe:** `recommendationStatus: not-evaluated`
- **Unsafe:** `status: delivered`
  **Safe:** `status: delivery-evidence`
- **Unsafe:** `overallStatus: accepted`
  **Safe:** `heuristicQualityStatus: accepted` + `confidenceSource: application-heuristic`
- **Unsafe:** `inControl: true`
  **Safe:** `status: not-evaluated` (unless signals calculated)
