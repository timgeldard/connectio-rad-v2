# Reference-consumer contract-alignment audit

> **Scope**: catalogue of frontend / domain-integration consumers for the
> eight data products that recently moved to source-truthful unavailable
> states (PRs #102, #105, #108, #109). Each row records whether the
> consumer preserves governed `null` / `unknown` / `unavailable` /
> `not-evaluated` semantics or silently converts them into reassuring
> defaults.

## Method

Grepped the eight data product field names, the forbidden literal labels
listed in the brief Part D, and the `String(... ?? '')` / `Number(... ?? 0)`
patterns across `domain-integrations/*/src/`. Verified each match against
the relevant Zod schema in `packages/data-contracts/src/schemas/`.

## Summary of changes

This PR modifies three files:

1. `domain-integrations/warehouse/src/adapters/warehouse-360-legacy-api-adapter.ts` — 4 mapping blocks (inbound, outbound, staging, exceptions) now preserve `null` for source-absent fields and `null` for unrecognised severity.
2. `domain-integrations/operations/src/adapters/process-order-review-legacy-api-adapter.ts` — `orderStatus`, `plannedStart`, `plannedFinish`, `actualStart`, `actualFinish` no longer fall back to `'released'` / current date; they fall back to `'unknown'` / `null` per the source-truthful contract.
3. `domain-integrations/traceability/src/trace-app/MassBalancePanel.tsx` — `Kpi` component prop type widened to accept `string | null` and renders `'—'` for a missing source UOM instead of letting a caller invent `'KG'`.

Plus tests + this audit doc.

## What was deliberately NOT changed

- Trace customer-exposure / customer-deliveries panels — `&&`-gated render of `recallRecommended` is already null-safe.
- POH header panel — `STATUS_COLOR[orderStatus] ?? …` already falls through safely for `'unknown'`.
- WarehouseReconciliationExceptionsPanel — uses a different (mock-only) schema and is not affected by the recent nullability changes.
- SPC adapter UOM — already emits `null` (PR #105).
- Trace mock data — mock scenarios legitimately assert governed-source pretend values where the mock is simulating a governed source.
- The `'G' → 'KG'` unit conversion in `order-history-view.tsx` — it is a legitimate UOM conversion, not a default-when-missing.
- All `String(... ?? '')` patterns in the POH legacy adapter for fields where the contract still requires `string` (`uom`) or `number` (`plannedQuantity`, `confirmedQuantity`). Flagged as a follow-up for a future contract relaxation PR.

## Follow-ups tracked

- POH `uom`, `plannedQuantity`, `confirmedQuantity` contract relaxation (mirror of PR #105 SPC UOM pattern).
- `STATUS_COLOR` explicit `'unknown'` entry (documentation).
- Migrate `WarehouseReconciliationExceptionsPanel` from the legacy schema to `Warehouse360ExceptionItem` — or formally retire the legacy schema.
